# LCSAJdump v1.2.0 — Changelog

## Summary

Version 1.2.0 introduces four new CLI options, a reworked output pipeline, and significant improvements to core flexibility. The old `-f/--file` flag (which saved to a hardcoded `gadgets_found.txt`) has been removed and replaced with a more flexible `-o/--output PATH` option.

---

## CLI Changes

### Removed

| Option | Reason |
|---|---|
| `-f, --file` | Replaced by `-o/--output`. The hardcoded output filename `gadgets_found.txt` was inflexible for scripting and pipeline usage. |

### Added

| Option | Type | Description |
|---|---|---|
| `-o, --output PATH` | PATH | Write gadgets to a custom file path. Works with both plain text and `--json` output. |
| `-b, --bad-chars TEXT` | TEXT | Provide a hex string of bytes to filter from gadget addresses (e.g. `"000a0d"` filters null bytes, newlines, and carriage returns). |
| `--json` | FLAG | Output gadgets as structured JSON instead of plain text. Suitable for programmatic consumption and integration with other tools. |
| `--all-exec` | FLAG | Analyze all executable ELF sections (e.g. `.init`, `.plt`, `.fini`) in addition to `.text`. Useful for finding gadgets in non-standard section layouts. |

### Present in v1.1.3.1 but now documented

| Option | Type | Description |
|---|---|---|
| `-i, --instructions INTEGER` | INTEGER | Max number of instructions per LCSAJ node. Controls how coarse-grained the graph nodes are. Default: `15`. |

---

## Core Changes

### `BinaryLoader`
- Now accepts `all_exec=True/False` parameter.
- When `all_exec=True`, all ELF sections with the `SHF_EXECINSTR` flag are loaded and concatenated for disassembly, rather than only `.text`.

### `RainbowFinder`
- Now accepts `max_insns` as a constructor parameter (previously hardcoded).
- New method: `gadgets_to_json(limit, min_score, bad_bytes)` — serializes the ranked gadget list to a JSON string with fields: `address`, `instructions`, `score`, `type`.

### Output Summary
- Summary line now uses structured dict: `summary['total']`, `summary['shown_seq']`, `summary['shown_jmp']`.
- `bad_bytes` is now propagated to both `print_gadgets()` and `gadgets_to_json()`.

---

## Migration Guide

### From v1.1.3.1 to v1.2.0

| Old command | New equivalent |
|---|---|
| `lcsajdump -f <binary>` | `lcsajdump -o gadgets_found.txt <binary>` |
| `lcsajdump -a x86_64 -f <binary>` | `lcsajdump -a x86_64 -o gadgets_found.txt <binary>` |
